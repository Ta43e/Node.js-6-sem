1.ORM расшифровывается как Object-Relational Mapping. Это паттерн программирования, который позволяет связывать объекты в приложении напрямую с записями в базе данных,
 обеспечивая абстракцию от специфики работы с базами данных. ORM позволяет разработчикам работать с базой данных, используя объектно-ориентированный подход, вместо традиционных SQL-запросов.

2.Для работы Sequelize с Microsoft SQL Server необходимо установить дополнительный пакет mssql.
 Этот пакет позволяет Sequelize взаимодействовать с базой данных SQL Server.

3.В контексте Sequelize и баз данных, модель представляет собой абстракцию данных,
 которая соответствует определенной таблице в базе данных. Модель определяет структуру данных, которые будут храниться в таблице, а также методы доступа и манипуляции этими данными.

4.Для выполнения SELECT-запросов в Sequelize используется метод findAll().
 Этот метод позволяет найти все записи, которые соответствуют определенным критериям, в соответствии с моделью, связанной с базой данных.

5. Для выполнения INSERT-запросов в Sequelize используется метод create().
 Этот метод создает новую запись в базе данных на основе переданных данных и модели, связанной с базой данных.
